## 한 줄 정체성

**기존 MBTI가 "공감되지 않는다"는 문제를 정조준한, 공감되는 한국식 성격 테스트.**

기존 MBTI는 칼 융 심리학에 뿌리를 둔 학술적 분류라, J/P 차이를 따로 공부해야 알 정도로 일상 직관과 멀다. 그래서 결과를 받아도 "정말 내가 이게 맞나?" 싶고, 친구에게 보여주면 "너가 I라고?"라는 반응이 돌아오기 일쑤다. 우리는 학술적 정확성 대신 **"느낌이 맞다"**는 공감을 목표로 한다.

## 페르소나 — 누가 이걸 좋아할까

인구통계(나이/성별/직업)로 좁히지 않는다. 대신 **콘텐츠/기존 MBTI와의 관계**로 정의한다.

### 1. 결과가 "내 얘기"여야 만족하는 사람들
- 자기 분석·자기 이해에 관심이 많다
- "어떻게 이걸 알았지?" 싶은 디테일에 반응한다
- 결과 페이지를 캡처해서 카톡/인스타에 올리는 게 자연스럽다

### 2. 기존 MBTI에 미묘한 불만이 있는 사람들
- "내가 정말 ENFP가 맞나?" 하고 결과를 의심해본 적이 있다
- "J랑 P는 도대체 뭔 차이야?" 식으로 카테고리 자체에 답답함을 느낀다
- 주변에서 "너가 I라고?", "너가 T라고?" 같은 의외라는 반응을 들어본 적이 있다
- 결과 글이 너무 길고 일반론이라 끝까지 안 읽어본 적이 있다

### 3. 콘텐츠를 "화제거리"로 쓰고 싶은 사람들
- 회식·소개팅·단톡방에서 분위기 띄우는 도구로 쓴다
- 친구 결과와 비교하면서 노는 게 본 목적이다
- "이거 딱 너야", "이거 우리 팀장이야" 식으로 태깅하고 싶다

## 제품 컨셉 — 카드 3장 모자이크

**한 사람 = 16개 칸 중 하나**가 아니라, **한 사람 = 카드 3장의 조합**.

```
🃏 연애 카드  ──┐
🃏 가족 카드  ──┼──→  "롬-콤 한복판 K-장녀 효율러"
🃏 공감 카드  ──┘
```

각 카드는 서로 다른 도메인을 진단한다. 친구와 비교하면 카드 하나는 같고 둘은 다를 수 있어, **단톡방 대화 거리**가 자연스럽게 생긴다.

### 왜 16칸이 아닌가
- MBTI 16칸은 16Personalities·푸망·msbti가 이미 다 점유. 후발주자로 들어가면 "또 다른 MBTI"가 된다.
- 16칸 단일 라벨은 자기서사를 한 단어로 압축해버려, 결과가 "어디서 본 것 같다"가 되기 쉽다.
- 카드 3장 × 각 약 5~8종 = **조합 100개+**. 친구 결과와 미묘하게 어긋나는 게 viral의 연료다.

## 핵심 의사결정 4가지 + 트레이드오프

### 결정 1 — 분류 구조: 3카드 모자이크

| 옵션 | 장점 | 단점 |
|---|---|---|
| (A) MBTI식 16칸 단일 라벨 | 익숙함, 구현 단순 | 짝퉁 인상, 시장 포화 |
| **(B) 3카드 모자이크** ✅ | 새 게임, 조합 다양성, 단톡방 대화거리 | 결과 페이지 설계 복잡, 사용자 학습 비용 |
| (C) Big Five식 연속 점수 | 학술 정확 | 카드화 어려움, viral 실패 전례 (카카오 마음날씨 8년 600만 명에도 viral 폭발 없음) |

**선택 이유:** MBTI 짝퉁이 되지 않으면서도 한 축이 아닌 여러 축으로 자기서사를 만들 수 있는 유일한 구조. 복잡도 증가는 결과 페이지 UX로 흡수.

### 결정 2 — 카피 톤: 양극단 캐릭터화

중간 50%는 **일부러 시시하게**, 양극단 25%씩만 **진하게** 쓴다.

```
EQ 0~20  ──→  T발놈 효율러       (진하게)
EQ 21~50 ──→  적당히 공감하는 사람 (시시하게)
EQ 51~80 ──→  공감과부하 흡수형   (진하게)
```

**왜 중간을 일부러 약하게 쓰나:**
- viral로 퍼지는 카피는 항상 극단 묘사다. "와 이거 완전 내 얘기"는 평균치에서 안 나온다.
- 양극단을 진하게 쓰면 그 대비로 양극단 캐릭터가 더 도드라진다.
- 검증된 패턴: 푸망 T발놈 테스트, K-장녀 vs 외동 밈 모두 양극단 캐릭터화.

**트레이드오프:** 중간에 떨어진 사용자의 만족도는 낮아진다. 그러나 viral 전파력의 핵심은 끝단이라 의도된 손실.

### 결정 3 — 학술 면책: 백엔드 학술 / 프론트 밈

**진단 계산은 학술 척도로, 사용자 화면에는 학술 흔적 0.**

| 영역 | 사용자 안 봄 (백엔드) | 사용자 봄 (프론트) |
|---|---|---|
| 연애 | Sternberg TLS-15 (37개 언어 cross-cultural 검증, α 0.86~0.96) | "롬-콤 한복판러" |
| 공감 | Baron-Cohen EQ K-EQ (Kim & Lee 2010, α 0.78) | "T발놈 효율러" |
| 가족 | Adler 출생순위 (학술은 약하지만 K-장녀 viral 확인) | "K-장녀" |

**왜 학술을 숨기나:**
- 카카오 마음날씨가 Big Five 학술 척도로 8년 운영, 600만 명 사용. 그러나 viral은 안 터졌다. "학술 검사" 인상이 강하면 MZ가 농담으로 안 받아들이고, 친구한테 안 보낸다.
- EQ는 Baron-Cohen 자폐 진단 도구에서 출발 — 사용자에게 노출되면 신경다양성 백래시 위험. **학자 이름·논문·자폐 컷오프 같은 어휘는 마케팅 메시지에 절대 노출 X.**

**트레이드오프:** "근거 있는 테스트"라는 신뢰 차별화를 못 쓴다. 그 대신 viral 동력의 면책("재미로 보세요")을 얻는다. 학술은 우리의 무기가 아니라 안전망.

### 결정 4 — 3카드 동시 출시 + 우선순위 다듬기

현재 [02_fe_react/src/results/INFP.tsx](02_fe_react/src/results/INFP.tsx)는 MBTI 16칸 구조의 잔재. **3카드 구조로 갈아엎는다.**

```
src/results/
├── cards/
│   ├── love/          ← Sternberg 8유형
│   │   ├── Romantic.tsx
│   │   ├── Empty.tsx
│   │   ...
│   ├── family/        ← 출생순위 5유형
│   │   ├── KEldest.tsx
│   │   ...
│   └── empathy/       ← EQ 4구간
│       ├── TBaller.tsx
│       ├── Absorber.tsx
│       ...
└── ResultPage.tsx     ← 카드 3장 조합 렌더
```

**셋 다 동시 출시.** 검사 종료 → 카드 3장이 노출됨 → 사용자가 하나씩 클릭해서 결과를 확인하는 동선.

**카피 다듬기 우선순위: 연애 > 가족 > 공감.**
- 직관 근거: 사용자 관심도 추정 (연애가 가장 높고, 공감이 가장 낮다고 가정). 학술 검증은 안 거침 — 심심풀이 프로젝트라 직관으로 출발.
- 출시 후 카드별 자발적 공유율로 직관이 맞는지 자연실험 측정 → 자원 재배분 가능.
- 작가 자원·반복 수정은 연애 카드에 가장 많이 투입.

**트레이드오프:**
- (vs MVP 1카드 안) 출시 지연 — 셋 다 빌드 + 카피 작성이라 일정 길어짐. 대신 SPOF(한 카드 viral 실패 = 프로젝트 실패) 분산 + 카드별 viral 적합도 자연실험.
- 연애 카드 자기배제 위험 — Sternberg는 연애 경험·관심을 전제. 솔로·무성애자·중고생 사용자의 "내 얘기 아님" 반발 가능. "이상적 사랑관"으로 우회하면 학술 매핑이 흔들리니 카피 단계에서 처리.

## 이론 backbone — 카드별 선정 이유

15개 심리학 이론을 [.claude/docs/02_psy/](.claude/docs/02_psy/)에서 비교 분석. 7개 기준(분류 명확성·공감 친화성·밈 잠재력·학술 신뢰도·MBTI 차별화·한국 시장·viral 사례)으로 채점.

| 카드 | 채택 이론 | 채택 이유 | 탈락 후보 |
|---|---|---|---|
| 연애 | **Sternberg 사랑의 삼각형** | 8유형 카드화 명확, TLS-15 한국어 검증, viral 미개척 블루오션 | Attachment Style(이미 viral 포화), Love Languages(라이선스) |
| 가족 | **Adler 출생순위** | K-장녀 밈 이미 viral 확인, 한국 가족 문화와 직결 | Schwartz Values(학술색 무거움) |
| 공감 | **Baron-Cohen EQ (구간화)** | T발놈을 학술 우회 가능, 양극단 캐릭터화 최적 | HEXACO(부정 라벨 위험), Big Five(viral 실패 전례) |

상세 비교는 [.claude/docs/02_psy/00_overview.md](.claude/docs/02_psy/00_overview.md).

## 수익화 모델

**트래픽 → 광고**가 본업. 학술 백엔드와 viral 프론트의 분리가 트래픽 확보를 위한 설계.

- 결과 페이지 노출 광고
- 트래픽 일정 임계치 이상에서 도입 (지금은 viral 검증 우선)
- 유료화·구독 모델은 검토 안 함 (viral 마찰 증가)

## 다음 단계

1. **3카드 문항 설계** — 연애(Sternberg TLS-15) / 가족(출생순위) / 공감(K-EQ) 백엔드 학술 척도 매핑.
2. **결과 카피 양극단 캐릭터화** — 카드 종류별 진한 묘사 / 시시한 묘사 분리. **우선순위: 연애 > 가족 > 공감.**
3. **결과 페이지 UX** — 검사 종료 → 카드 3장 노출 → 클릭으로 하나씩 공개되는 동선.
4. **친구 공유 기능 빡세게** — 카톡 공유 / 인스타 스토리 / URL 복사 / 결과 비교 / referral 추적. MVP 확산이 지인 채널로 제한되니까 사용자당 viral 계수를 최대화하는 게 전략. 결과 페이지의 핵심 동선.
5. **공유 OG 이미지** — 카드별 캡처 친화 디자인 (카드 1장 단위가 viral 공유 단위).

## MVP 확산 전략

**초기 시드 트래픽은 지인 채널로 확보한다.** 광고·인플루언서 시딩은 viral 검증 전엔 보류.

- 지인 카톡 단톡방
- 인스타 (스토리·피드)
- 유튜브 (직접 콘텐츠 제작해 트래픽 유도)

전제: viral 자체 동력이 약해도 분배 채널을 본인이 갖고 있으므로 0 → 1 문제는 외부 변수가 아님. 그 대신 **사용자당 친구 호출 효과를 극대화**하는 데 제품 자원을 집중한다 (다음 단계 4번).

## 디자인 원칙 — 모바일 우선

**대다수 사용자가 모바일에서 접근한다.** 폰트 사이즈와 레이아웃 배치에 굉장히 힘을 써야 한다. 데스크톱은 깨지지 않는 수준만 유지하고, UX 디테일은 모바일 기준으로 빡세게 다듬는다.

- **폰트**: 모바일에서 가독성 확보되는 사이즈가 기준. 본문 16px 이상, 라벨·헤딩은 비율로 키움. 작아 보이면 무조건 키운다.
- **배치**: 9:16 세로 화면에서 한 손 엄지로 스크롤하며 결과를 흡수할 수 있는 흐름. 결과 페이지는 모바일 한 스크롤 캡처가 인스타 스토리에 그대로 박힐 수 있어야 함.
- **터치 타깃**: 버튼·링크 44px+, 패딩 넉넉히. 손가락이 닿는 모든 요소는 모바일 손가락 기준.
- **검증**: 데스크톱 dev 환경에서 멋있어 보여도 모바일에서 안 예쁘면 실패. 모든 UI 변경은 모바일 실기기 확인 후 머지.

docker compose --profile dev up -d --build
