## 한 줄 정체성

**카드 4장으로 보는 연애·가족·친구·직장/학교에서의 나의 모습**

사람은 외부에서 내가 어떻게 보이는지에 대한 궁금증이 항상 있다. 누군가 '나만의 보석'을 찾아주기를 몹시 바란다.

기존의 MBTI 같은 성격 유형 검사는, 상황에 따라 사람이 입체적이라는 사실을 배제한다. 우리 서비스는 상황에 따라 사람이 충분히 달라질 수 있음을 인정하고, 네 가지 상황에 따라서 사람의 성격을 정의한다.

기존 MBTI 는 위와 같은 이유에서 "대충 무슨 말인지는 알겠는데, 공감하기 어렵고 일반론적이다"는 평. 또한, 학술적인 분류라, N/S 차이를 따로 공부해야 알 정도로 일상의 직관과 벌다.

그래서 결과를 받아도 "정말 내가 이게 맞나?" 싶고, 친구에게 보여주면 "너가 I라고?"라는 반응이 돌아오기 일쑤다. 우리는 학술적 정확성 대신 **"느낌이 맞다"**는 공감을 목표로 한다.

## 페르소나 — 누가 이걸 좋아할까

인구통계(나이/성별/직업)로 좁히지 않는다. 대신 **콘텐츠/기존 MBTI와의 관계**로 정의한다.

### 1. 결과가 "내 얘기"여야 만족하는 사람들
- 자기 분석·자기 이해에 관심이 많다
- "어떻게 이걸 알았지?" 싶은 디테일에 반응한다
- 결과 페이지를 캡처해서 카톡/인스타에 올리는 게 자연스럽다

### 2. 기존 MBTI에 미묘한 불만이 있는 사람들
- "내가 정말 ENFP가 맞나?" 하고 결과를 의심해본 적이 있다
- "J랑 P는 도대체 뭔 차이야?" 식으로 카테고리 자체에 답답함을 느낀다
- 주변에서 "너가 I라고?", "너가 T라고?" 같은 의외라는 반응을 들어본 적이 있다
- 결과 글이 너무 길고 일반론이라 끝까지 안 읽어본 적이 있다

### 3. 콘텐츠를 "화제거리"로 쓰고 싶은 사람들
- 회식·소개팅·단톡방에서 분위기 띄우는 도구로 쓴다
- 친구 결과와 비교하면서 노는 게 본 목적이다
- "이거 딱 너야", "이거 우리 팀장이야" 식으로 태깅하고 싶다

## 제품 컨셉 — 카드 4장 모자이크 (모드 기반)

**한 사람 = 16개 칸 중 하나**가 아니라, **한 사람 = 상황별 4가지 모습의 조합**.

```
🃏 연애에서의 나의 모습        ──┐
🃏 가족끼리의 나의 모습        ──┼──→ "롬콤 한복판러 / K-장녀 / 친구 광대 / 회사 빌런"
🃏 친구끼리의 나의 모습        ──┤
🃏 직장(학교)에서의 나의 모습  ──┘
```

각 카드는 서로 다른 상황을 진단한다. 친구와 비교하면 카드 중 일부는 같고 일부는 달라 **단톡방 화제거리**가 자연스럽게 생긴다.

### 왜 "~에서의 / ~끼리의 나의 모습" 인가

- *나*를 다양한 상황에서 본다 → "같은 사람의 4면" 자기서사. 카드 간 대비가 viral 연료다. ("회사에서는 T인데 연애에서는 F잖아")
- **친구 카드 메타 모순 해소** — "친구로서의 너"가 아니라 "친구끼리의 *나의* 모습"이라 친구한테 공유가 자연스럽다. "야 너 우리 만날 때 진짜 이러잖아".
- **자기배제 회피** — 모태솔로도 "내가 연애한다면"으로 진입 가능. 학생도 "직장(학교)" 병기로 학교 모드 매핑. 무직·구직·프리랜서는 카피 단계에서 "주된 활동 공간" 류로 한 줄 인정.

### 왜 16칸이 아닌가

- MBTI 16칸은 16Personalities·푸망·msbti가 이미 다 점유. 후발주자로 들어가면 "또 다른 MBTI"가 된다.
- 16칸 단일 라벨은 자기서사를 한 단어로 압축해버려, 결과가 "어디서 본 것 같다"가 되기 쉽다.
- 카드 4장 × 각 약 5~8종 = **조합 수백 단위**. 친구 결과와 미묘하게 어긋나는 게 viral의 연료다.

## 카피 톤 — "너의 보석을 찾아주겠다"

**장점은 진하게, 단점은 유머로 승화한다.** 중간 50%는 일부러 시시하게, 양극단 25%씩만 진하게.

| 차원 | 톤 가이드 |
|---|---|
| 중간(평범) | 일부러 시시하게 — viral 동력은 끝단 |
| 양극단(장점) | 진하게 — "이거 진짜 내 매력" 자랑하고 싶음 |
| 양극단(단점) | 유머로 승화 — "내 단점도 이렇게 귀엽게 봐주네" |

**왜 부정 단독 라벨(푸망 T발놈 류)을 안 쓰는가:**
- BuzzFeed "결과는 항상 긍정" 법칙 — 공유율 가장 높은 톤.
- *낙인형* 부정 라벨은 받은 사람 기분 상함 → 1회성 공유로 끝남.
- "너의 보석" 톤은 *나의 다른 면 보석*을 찾아 4카드 다 클릭하게 만든다 → 재이용 동력.

**가드레일:**
- 단점을 너무 부드럽게 처리하면 "다 좋은 말만 한다"는 짭짤한 맛 부족. 유머 강도는 빡세게 조절.
- "보석"이 *온실 칭찬*이 되면 안 됨 — 구체적 일상 장면 + 자기인지 가능한 표현이라야 공감.
- 부정 면은 *상황 코미디*로만 노출 ("니가 그러는 이유가 있긴 함" 류).

**트레이드오프:** 중간에 떨어진 사용자의 만족도는 낮아진다. 그러나 viral 전파력의 핵심은 끝단이라 의도된 손실.

## 심리 이론은 중요하지 않다

학술 이론은 우리의 **무기가 아니라 안전망**이다. 사용자 화면에는 학술 흔적 0, 면책은 곳곳에.

### 사용자 화면에는 학술 흔적 0

학자 이름·논문 제목·이론명·척도 약어는 사용자가 보는 결과지·문항·라벨·소개 어디에도 노출되지 않는다. 진단 계산이 어떤 학술 척도를 *백엔드*로 쓰든, 그건 사용자가 모르는 안전망일 뿐.

학술색이 강하면 MZ는 농담으로 안 받아들이고 친구한테 안 보낸다. 카카오 마음날씨가 Big Five 학술 척도로 8년 운영 600만 명에도 viral 안 터진 이유.

### 학술 매핑은 미루고, 카피가 먼저

학술 척도(연애의 어떤 이론, 가족의 어떤 이론…)는 **카피가 박힌 다음 역방향으로** 결정한다. 학술을 먼저 정하면 카피가 학술 어휘에 *눌리고*, "재미로 보세요" 면책이 깨진다. 현재는 연애 카드만 학술 매핑이 잠정 잡혀 있고, 나머지 3카드는 *카피 떨어진 다음 결정*.

### 면책은 곳곳에 박는다

"재미로 보세요" 류 면책은 한 곳에 모아두지 않고 사용자 동선 곳곳에 박는다:
- 검사 시작 화면
- 결과지 머리말 / 꼬리말
- 공유 OG 이미지 카피
- 친구 비교 페이지

학술 검사로 *오인*되는 순간 viral 동력은 죽는다.

### 트레이드오프

"근거 있는 테스트"라는 신뢰 차별화를 못 쓴다. 그 대신 viral 동력의 면책을 얻는다. 학술은 우리의 무기가 아니라 안전망 — 자기배제 회피, 분류 정합성 정도의 안전 장치.

## 수익화 모델

**트래픽 → 광고**가 본업. 학술 백엔드와 viral 프론트의 분리가 트래픽 확보를 위한 설계.

- 결과 페이지 노출 광고
- 트래픽 일정 임계치 이상에서 도입 (지금은 viral 검증 우선)
- 유료화·구독 모델은 검토 안 함 (viral 마찰 증가)

---

## MVP 확산 전략

**초기 시드 트래픽은 지인 채널로 확보한다.** 광고·인플루언서 시딩은 viral 검증 전엔 보류.

- 지인 카톡 단톡방
- 인스타 (스토리·피드)
- 유튜브 (직접 콘텐츠 제작해 트래픽 유도)

전제: viral 자체 동력이 약해도 분배 채널을 본인이 갖고 있으므로 0 → 1 문제는 외부 변수가 아님. 그 대신 **사용자당 친구 호출 효과를 극대화**하는 데 제품 자원을 집중한다.

## 디자인 원칙 — 모바일 우선

**대다수 사용자가 모바일에서 접근한다.** 폰트 사이즈와 레이아웃 배치에 굉장히 힘을 써야 한다. 데스크톱은 깨지지 않는 수준만 유지하고, UX 디테일은 모바일 기준으로 빡세게 다듬는다.

- **폰트**: 모바일에서 가독성 확보되는 사이즈가 기준. 본문 16px 이상, 라벨·헤딩은 비율로 키움. 작아 보이면 무조건 키운다.
- **배치**: 9:16 세로 화면에서 한 손 엄지로 스크롤하며 결과를 흡수할 수 있는 흐름. 결과 페이지는 모바일 한 스크롤 캡처가 인스타 스토리에 그대로 박힐 수 있어야 함.
- **터치 타깃**: 버튼·링크 44px+, 패딩 넉넉히.
- **검증**: 데스크톱 dev 환경에서 멋있어 보여도 모바일에서 안 예쁘면 실패. 모든 UI 변경은 모바일 실기기 확인 후 머지.

## 기술 스택

- **FE**: React 18 + TypeScript 5 + Vite 5
- **Styling**: CSS Modules + 전역 `styles/global.css`
- **Proxy**: nginx 1.27-alpine
- **Orchestration**: Docker Compose · `--profile feature` (HMR) / `--profile dev` (빌드 스냅샷)
- **Host**: `*.atchu.co.kr`

```bash
# 작업 중 (HMR · http://localhost:4080)
docker compose --profile feature up -d --build

# 빌드 스냅샷 (http://localhost:4090)
docker compose --profile dev up -d --build
```
