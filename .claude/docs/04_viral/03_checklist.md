# 실용 체크리스트 — 사이드 프로젝트 기획·디자인에 바로 적용

리서치 결과를 기획·구현에 바로 쓸 수 있는 형태로 정리. 라운드 2.5 반박이 반영된 보수적 가이드라인.

---

## 1. 결과 페이지 카피 (가장 중요 — 작가 역량이 차별화)

### 필수 5단 구조 (가이드라인, 검증된 공식 아님)

```
[1단] OO한 당신,
       또는 "당신은 ~한 사람이에요"

[2단] 양면 단정 — "겉으론 ~인데 속은 ~"
                 "~처럼 보이지만 사실은 ~"
                 "평소엔 ~하다가 ~할 때만 ~"

[3단] 단점의 귀여운 라벨화 (사용자의 자기서사 활용)
      - 무뚝뚝 → "쓸데없는 말 안 하는 효율러"
      - 예민 → "남보다 한 박자 먼저 느끼는 사람"
      - 게으름 → "에너지 효율 1등급"
      - 욕설 라벨(T발놈류)은 자기조롱이 허용되는 한도 내에서만

[4단] 위로형 결말 — "그래도 당신은 충분히 ~"
                  "지금 그대로도 괜찮아요"
                  "이런 사람 곁에 두면 행운이에요"

[5단] 공유 유도 — 궁합 유형 표시 + "친구는 뭐 나왔어?" 버튼
```

### 글쓰기 가드레일 (라운드 2.5 반영)

- ✅ 단점을 **자기조롱 가능한 밈**으로 번역 (사용자 본인이 카톡에 자기 결과 붙이고 웃을 수 있어야 함)
- ❌ **외부에서 차별 도구로 쓰일 라벨 금지** ("이런 사람과는 결혼 X" 같은 비교 라벨 회피)
- ❌ **진지·교훈적 톤 금지** ("당신은 자기 자신을 더 사랑해야 합니다" 같은 라이프코칭 톤)
- ✅ **양면성 활용** — 한 면만 강조하면 "나는 그런 사람 아닌데" 반발. 양면 다 긍정해야 받아들임
- ✅ **구체적 디테일** — "당신은 외향적입니다" (X) vs "처음 만난 사람한테도 어색한 침묵을 못 견뎌서 결국 본인이 먼저 말 거는 타입" (O)

### 한 줄 핵심 — "공감"의 정의

> **"내가 일상 카톡에서 농담으로 쓰던 자기서사를 누가 대신 라벨링해주는 경험"**
> 정확한 진단 ≠ 공감. 한국식 viral test의 진짜 차별점은 이거.

## 2. 결과 카테고리 설계

### 권장 (가이드라인)

- **카테고리 수**: 16개 표준이지만 절대 최적 X. 차별화로 다른 수(SBTI 29개 사례) 시도 가능.
- **각 결과는 한 단어 라벨 + 캐치프레이즈 1줄**로 압축 가능해야 함 (카톡/스토리에 한 줄 캡션으로 떠다닐 형태).
- **궁합 유형**을 결과마다 표기 → 친구 호출 동선 확보.
- **캐릭터/동물 매칭**은 보너스. 매칭 정확도·공감 없이 그냥 매칭만 하면 안 됨(미쏘 실패 사례).

### 주의

- 카테고리 수가 너무 적으면(4개 이하) "내 얘기 같지 않다" 반발 — 혈액형보다 16개 우세 이유
- 너무 많으면(50개+) 결과 페이지 콘텐츠 양산 부담 + 사용자 검색·비교 어려움
- **균형점**: 12~24개 사이 + 친구와 비교 가능한 구조

---

## 3. 질문 설계

### 권장 가이드라인

- **문항 수**: 12~20개 (한국 viral test 평균, ktestone·poomang 관찰 기반)
  - 글로벌 표준 6~8개는 너무 짧음 — "정밀해 보이는 게 신뢰 줌"이 한국 특수성
  - MBTI 60+개는 이탈률 위험 — 사이드 프로젝트는 절충안
- **형식**: **상황형 우세** (자기보고형보다)
  - ❌ "나는 외향적인 편이다 (1~5점)"
  - ✅ "친구가 약속 30분 전에 갑자기 취소했다. 너의 솔직한 반응은?"
- **답안 수**: 2~4개 (5개 이상은 인지 부담)
- **이미지 선택형**은 보너스 옵션 — 시각 자극으로 인게이지먼트 ↑

### 모바일 최적화 필수

- 한 화면에 질문 1개 + 답안 버튼 (스크롤 X)
- 진행률 표시바
- 로딩 수초 이내

## 4. 결과 페이지 모바일 디자인

### 필수

- **9:16 캡쳐 친화 카드** — 인스타 스토리 직접 업로드 가능
- **모바일 한 스크롤 내** 결과 핵심(라벨+캐치프레이즈+본문 2문단+궁합) 다 들어감
- **결과별 고유 URL** — `/result/wnghkd` 식. 카톡으로 결과 자체 떠다님 (보조 기제이지만 트래픽 유입에 유효)
- **결과 직후 큰 공유 CTA** — "친구는 뭐 나왔을까?" 버튼
- **궁합 유형 카드** — 친구 태그 유도

### 카피 부분 길이

- 라벨명: 1단어
- 캐치프레이즈: 1줄 (15자 내외)
- 본문: 2~3문단 (150~300자)
- 궁합 유형: 1줄
- 추가: 행동 권유/위로 1줄

## 5. 공유 동선

### 필수 버튼 (결과 페이지)

1. 카톡 공유 (개별 친구 + 단톡방)
2. 인스타 스토리 공유 (이미지 자동 변환)
3. URL 복사 (트위터/X용)
4. 친구도 해보기 (테스트 처음으로 돌아가기 + 추적 가능한 referral URL)

### 권장 추가

- **친구 결과와 비교** 기능 — "내 OO 어디감" 캡션 패턴 자동 생성
- **결과 카드 이미지 다운로드** — 캡쳐 안 해도 됨

### 권장

- **트래픽 임계 도달 전엔 광고 없이** — 결과 페이지 경험 보호 (메모리 [monetization_model.md] 전략 그대로)
- 임계 도달 후:
  - **결과 페이지 광고 위치**: 결과 본문 위/아래 No, 본문 사이 No → 결과 본문 보고 난 후 **궁합/공유 CTA 아래**에 배치 (감정 절정 후)
  - **광고망**: 애드센스 + 카카오 애드핏 병행 (한국 트래픽 CPM 비교)

- **광고 외 수익 채널 고려**:
  - 결과별 굿즈/스티커 판매
  - 연애 상담·심리 상담 추천 어필리에이트
  - 브랜드 콜라보 (브랜드 캐릭터를 결과 카테고리로 활용)

## 7. 차별화 USP 메시지 (README와 일관)

### 시장 인식 활용 (라운드 2.5 확인)

- **시장은 이미 MBTI 피로감을 느끼고 있음**: Korea Times, Careet, 코스모폴리탄 보도 다수
- **사이드 프로젝트 USP는 정확한 방향**: "기존 MBTI가 공감 안 된다는 문제를 정조준" = 시장 니즈와 일치

### 메시지 전략

- ❌ "16가지보다 더 정확한 N가지" — 외형 차별화로는 식상감 백래시
- ❌ "MBTI보다 과학적" — 학술 정확성으로 승부하면 16Personalities에 못 이김
- ✅ **"기존 MBTI는 학술 분류라 일상이랑 멀다. 우리는 '느낌이 맞다'에 올인했다"** — 공감 톤·작가 역량으로 승부
- ✅ **"너가 카톡에 농담으로 쓰던 너 얘기를 결과로 돌려준다"** — 자기서사 라벨링 메시지

## 8. 출시 전 검증 (생존자 편향 회피)

라운드 2.5 강한 반박 — 패턴 따랐다고 viral 보장 X.

### 필수 검증 단계

1. **베타 단계 사용자 테스트** (N=20~30)
   - "이 결과가 내 얘기 같다" (1~5점)
   - "친구한테 보내고 싶다" (1~5점)
   - 두 항목 평균 4점 이상이어야 출시
2. **결과 페이지별 차이 측정** — 16개 결과 중 공감도 낮은 결과 식별 → 카피 재작성
3. **카톡 공유 실험** — 베타 사용자에게 카톡으로 결과 보내달라고 요청 → 실제 공유 캡션 관찰
4. **출시 후 결과별 공유율 추적** — Hotjar/Mixpanel로 결과 카테고리별 공유 버튼 클릭율 측정

### 안티 패턴 회피 체크

- [ ] 결과 페이지 진지·교훈적 톤 없음
- [ ] 결과 캐릭터가 자기인식과 어긋나지 않음 (미쏘 사례 회피)
- [ ] 차별·공격적 라벨 없음 (수협은행 사례 회피)
- [ ] 결과 URL이 결과별로 고유함
- [ ] 모바일 9:16 캡쳐 친화 디자인
- [ ] 친구와 비교 동선 있음
- [ ] 결과 직후 큰 공유 CTA

## 9. 한 줄 결론

> **차별화는 외형이 아니라 카피 톤에서 난다.** 16분류 vs N분류, 상황형 vs 자기보고형 같은 외형 선택은 표준 따라가도 무방. 진짜 차별화 포인트는 **"단점을 자기조롱 가능한 밈으로 번역하는 카피 작가 역량"** — 5단 공식은 가이드라인일 뿐, 한 줄 한 줄의 작가 감각이 viral과 묻힘의 경계다.
