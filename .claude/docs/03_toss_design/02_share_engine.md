# 02. 토스 결과 화면 패턴 + 우리 페이지의 공유 엔진 설계

> "어떻게 친구 사이에 더 많이 퍼지고 화제가 될 수 있는지" 의 답.
> 토스의 결과/리포트 화면 패턴을 분해하고, 우리 결과 페이지에서 공유를 유발할 4가지 트리거로 재조립.

---

## A. 토스의 결과/리포트 화면 패턴 분해

### A.1. 신용점수 리포트 [S19]

토스가 "단순 점수 조회"를 "**무료 요인 분석 리포트**"로 끌어올린 대표 사례.

**화면 위계**:
1. **최상위**: 점수 자체 (큰 숫자, 압도적 크기)
2. **차상위**: 긍정 요인 3개 / 부정 요인 3개 — **대조 구조 6개**
3. **하위**: 마이데이터 연동 시 추가 항목

**바이럴 메커니즘**:
- 점수 변동 시 **알림 발송** → 결과가 1회성 아닌 **재방문 트리거**
- 친구와 점수 비교 동기 자체가 강력 (자기 점수에 대한 호기심)

### A.2. 머니 리포트 (자산 분석/소비 리포트) [S20, S7]

**핵심 패턴**:
- **포인트 색상 1개**로 핵심만 강조 ("가장 많은 소비"·"붉은 알림 배지")
- 카테고리별 막대그래프 / 향후 예상 소비 / 순자산 추이
- "도전, 아껴 쓰기", "거래 횟수 top 5" 식 **아이콘 + 인포그래픽**으로 풀어내 심리적 진입장벽 낮춤

**바이럴 메커니즘**:
- "내가 이번 달에 ○○에 ○○만원 썼다" 같은 **수치의 충격성**이 캡처 욕구 자극
- 카테고리별 의외성 ("내가 이렇게 카페에 많이 썼다고?")

### A.3. 보험 진단 결과 [S1]

**핵심 패턴**:
- 결과 자체보다 결과로 가는 입력 단계의 **그래픽 안내**가 특징
- 결과 페이지에 항상 **다음 행동 CTA** (서류 업로드 등)

### A.4. 공통 룰

토스 결과 화면 3개에서 공통적으로 발견되는 패턴:

1. **수치/유형 1개를 압도적으로 강조** → 한눈에 인식
2. **포인트 컬러 1개로 위계** → 시선이 자연스럽게 핵심으로
3. **항상 다음 행동 CTA** → 결과로 끝나지 않음
4. **알림/변동 트리거로 재방문 유도** → LTV 확대

## B. SNS 공유 친화 결과 화면 — 외부 사례

### B.1. SBTI / 한국형 MBTI 변종 사례 [S21]

외부 분석에서 도출된 **공유 유발 패턴**:
- 4글자 영문 코드 + 결과 포스터(이미지)
- **카카오톡 단톡방·인스타 스토리 공유 최적화**
- 직관적 유형명(CTRL, DEAD 등)이 SNS 공유에 적합
- "내 친구는 뭐 나왔지?" 비교 동기 유발

### B.2. 토스 OG 이미지 가이드 [S22]

토스 자사 가이드 발췌:
- **눈길을 과도하게 끄는 크기/디자인 텍스트 금지**
- 보이스톤에 맞는 문구
- 즉, OG 이미지는 **장식 최소화 + 핵심 한 줄**

## C. 우리 페이지의 공유 트리거 4종

토스 패턴 + SBTI 사례 + 메모리에 정리된 페르소나 분석을 종합해, 우리 결과 페이지에 박을 **공유 유발 장치 4가지**.

### C.1. 트리거 1 — 캡처 친화 화면 (Screenshot-First Design)

**원리**: 사용자가 결과 페이지를 캡처해서 카톡·인스타에 그대로 붙이는 행동을 **default 유저 플로우**로 본다. "공유하기 버튼"보다 캡처가 먼저 일어난다는 가정.

**설계 룰**:
1. 결과 페이지 첫 화면을 **9:16 비율(인스타 스토리)**과 **1:1 비율(인스타 피드)** 둘 다에서 한 프레임에 잡히게 설계
2. 한 프레임 안에 반드시: **킬러 라인 1줄 + 유형명 4글자 + 일러스트 1개**
3. 캡처 영역에 **워터마크/도메인** 작게 박아두기 (좌하단 "{서비스명}.com" 같은) — 출처 추적용
4. 화면 상단·하단 헤더/CTA가 캡처 시 잘리지 않게 **여백 확보**

**검증**: 만든 화면을 실제 폰으로 띄우고 스크린샷 떠봐서, 그 한 장만 보고도 "이거 무슨 테스트야?" 싶은지.

### C.2. 트리거 2 — OG 이미지 분리 제작 (링크 공유 최적화)

**원리**: 카톡·페이스북 링크로 공유할 때 자동 생성되는 미리보기 이미지. 이게 매력적이지 않으면 링크 공유 효과 0.

**설계 룰**:
1. **OG 이미지 2종** 동적 생성:
   - **1.91:1 (1200×630)** — 카카오톡 / 페이스북 / 트위터 링크 미리보기 표준
   - **1:1 (1200×1200)** — 인스타 스토리 링크 스티커, 카톡 일부 케이스
2. OG 이미지에 **유형 4글자 + 킬러 라인 + 일러스트** (캡처 화면 축약본)
3. 16(또는 64) 유형마다 **유형별 OG 이미지 자동 생성** — `og-image.tsx` 같은 동적 라우트
4. 토스 룰 따라 **장식 최소화** [S22]

**기술 메모**: Vite SPA에선 OG 이미지를 사전 렌더링해야 함. 옵션:
- (a) 정적 OG 이미지 16~64장을 빌드 타임에 미리 만들어서 `public/og/` 에 둠
- (b) 별도 OG 생성 엔드포인트 (Cloudflare Workers, Vercel Edge 등) — 추후 고려

### C.3. 트리거 3 — 친구 비교 후크 (Compare Hook)

**원리**: 페르소나 3번 ("회식·단톡방에서 화제거리로 쓰고 싶은 사람들")의 핵심 동기. **"우리 친구는 뭐 나왔어?"** 가 공유의 진짜 엔진.

**설계 룰**:
1. 결과 페이지 하단에 **"내 친구도 해보면 좋을 사람"** 섹션
   - "당신과 환장의 케미: ENFP" / "당신을 가장 답답해할 유형: ESTJ" 같은 매핑
   - 16(또는 64) 유형 간 케미/궁합 표 미리 준비
2. **공유 카피 자동 생성** — 공유 시 자동으로 들어가는 메시지:
   - 카톡: "나 INFP래. 너는?  → [링크]"
   - 인스타 스토리: "나는 INFP — 머릿속 시끄러운 사람" (이미지 위 텍스트)
3. 친구가 결과 받은 후 **"이 사람과의 케미"** 페이지로 다시 데려오기 (양방향 비교)
4. (장기) 단톡방 단위 **그룹 결과 보기** — 같은 단톡방에서 N명이 했을 때 분포 시각화. 가장 강력한 바이럴 동력.

### C.4. 트리거 4 — 의외성 카피 (Caption-Worthy Lines)

**원리**: 페르소나 1번("결과가 내 얘기여야 만족")과 페르소나 3번 둘 다를 만족시키려면, **카톡으로 친구한테 보내고 싶은 한 줄**이 결과에 박혀있어야 한다.

**설계 룰**:
1. 모든 유형 결과 페이지에 **"친구 태그 라인"** 섹션 — 캡처 시 자연스럽게 친구가 떠오르는 한 줄
   - 예: "이거 우리 ○○ 같은데?" / "이거 딱 너야"
2. 결과 카피 안에 **단독 한 줄로 캡처 가능한 킬러 문장 3~5개**
   - 너무 길면 캡처 안 됨, 1~2줄 안에 끝나야 함
   - 다크 배경 위에 **포인트 컬러로 강조** → 캡처 시 시선 가는 영역 명시
3. 카피 자체의 **공감 + 살짝 부정적인 자조** 균형 (한국 SNS 공유 정서)
   - ✓ "회식 끝나고 카톡 답 천천히 보내는 거, 좀 미안하긴 해요"
   - ✗ "당신은 사회적 상황에서 회복 시간을 필요로 합니다"

## D. 페르소나별 공유 동기 매핑

[memory: differentiation_strategy + 페르소나 3종]에 정리된 페르소나 3유형 각각의 공유 동기와, 우리가 어떻게 그 동기를 자극할지.

| 페르소나 | 공유 동기 | 우리가 박을 트리거 |
|---|---|---|
| **1. 결과가 "내 얘기"여야 만족** | "어떻게 이걸 알았지?" 자기 발견의 충격 → 캡처해서 SNS 자기소개 | C.1 캡처 친화 + C.4 의외성 카피 |
| **2. 기존 MBTI에 미묘한 불만** | "기존 결과랑 다르네, 이게 진짜 나야" 라고 친구한테 알리고 싶음 | C.4 의외성 카피 ("네가 알던 INFP랑 좀 다르지?") |
| **3. 화제거리로 쓰고 싶음** | "이거 딱 너야", 친구 결과랑 비교 | C.3 친구 비교 후크 (자동 카피, 케미/궁합) |

페르소나 3번이 가장 큰 바이럴 엔진. 가장 강하게 설계.

## E. 토스 디자인 원칙과 공유 트리거의 결합

토스 원칙이 어떻게 공유 트리거를 강화하는지 1대1 매핑:

| 토스 원칙 | 공유에 어떻게 작용하나 |
|---|---|
| **3초 룰** (A.1) | 인스타 피드 스크롤 중 0.5초 안에 시선 잡아야 캡처가 멈춰서 보게 됨 |
| **1 Thing / 1 Page** (A.2) | 캡처 1장에 메시지 1개만 들어가야 SNS에서 즉시 이해됨 |
| **첫 화면 임팩트** (A.3) | OG 이미지·캡처 첫 프레임이 강해야 클릭/저장 유발 |
| **잡초 제거** (A.4) | 카톡으로 보낼 때 길면 안 읽힘. 핵심만 |
| **일상어 + Casual** (B) | 친구한테 보낼 때 학술 용어면 손이 안 감. 캡처해서 단톡방에 못 던짐 |
| **감정 공감** (B Emotional) | 공감되면 캡처, 안 되면 그냥 닫음 |
| **포인트 컬러 1개** (C.1) | 캡처 시 시선이 어디로 가는지 명확 → 한 장면 = 한 메시지 |
| **하단 BottomCTA** (C.4) | "친구한테 보여주기" 항상 노출 → 캡처 후 공유 흐름이 끊기지 않음 |

## F. 공유 흐름 — End-to-End 시나리오

사용자가 우리 결과 페이지에 도달한 후 공유까지 가는 이상적인 플로우.

```
[결과 등장]
   ↓ 모션: 작은 폭죽/감정 보상 (D 모션 채택)
   ↓ 첫 화면: 킬러 라인 + 유형명 + 일러스트 (3초 룰)
   ↓
[사용자가 캡처]
   ↓ 9:16 인스타 스토리 비율로 한 프레임에 잡힘 (C.1)
   ↓ 워터마크/도메인 좌하단 (C.1)
   ↓
[사용자 행동 분기 1: 즉시 인스타 스토리]
   → 캡처 그대로 올림. 도메인 봐서 친구가 들어옴.
   → 들어온 친구는 도메인만 보고도 "이거 뭐야?" → OG 이미지로 끌림 (C.2)
   ↓
[사용자 행동 분기 2: 카톡으로 친구한테]
   → "친구한테 보여주기" 버튼 (B BottomCTA, A.5 Thumb Zone)
   → Sheet 뜨면서 카톡/인스타/링크 분기 (C.5 Sheet 패턴)
   → 자동 생성 카피 "나 INFP래. 너는?" (C.3)
   → 카톡 미리보기에 1.91:1 OG 이미지 (C.2)
   ↓
[친구 진입]
   → OG 이미지 강력해서 클릭
   → 결과 페이지 진입 시 "이 친구는 INFP래. 너는 뭐야?" 후크
   → 테스트 시작
   ↓
[친구도 결과 받음]
   → 결과 페이지 하단에 "○○과의 케미" 섹션 (C.3)
   → 양방향 공유 루프 활성화
```

**바이럴 계수 K가 1을 넘으면 자생적 트래픽 폭증.** 위 플로우의 4개 트리거를 모두 박아야 K > 1 노릴 만함.

## G. 공유 카피 템플릿 (한국 SNS 톤)

자동 생성 공유 카피의 템플릿. 유형마다 변주.

### 카카오톡 (텍스트 + 링크 미리보기)
```
나 {유형명}래. 너는?
👉 {도메인}/{유형코드}
```
또는 더 후크 강한 버전:
```
{킬러 라인 한 줄}
나 {유형명}래 ㅋㅋ 너도 해봐
👉 {도메인}
```

### 인스타 스토리 텍스트 오버레이
```
나는 {유형명}
"{킬러 라인 한 줄}"
```

### 단톡방 던지기 (가장 큰 바이럴 풀)
```
{킬러 라인}
이거 진짜 정확한 듯 ㅋㅋㅋ
다들 한번 해봐  →  {도메인}
```

### X(트위터) — 한국에선 비중 낮지만 인플루언서가 픽업하면 폭발
```
"{킬러 라인}"
나 {유형명} 나옴. 다른 MBTI 결과보다 훨씬 정확한 듯
{도메인}
```

## H. 우선순위 (어디서부터 박나)

작업 시간이 한정적이라면 이 순서로 박는다:

1. **C.1 캡처 친화 화면** — 가장 큰 효과. 모든 트리거의 베이스
2. **C.4 의외성 카피** — 카피만 잘 써도 절반 먹고 들어감
3. **C.2 OG 이미지** — 링크 공유의 전환율 좌우
4. **B BottomCTA "친구한테 보여주기"** — 공유 흐름 막힘 제거
5. **C.3 친구 비교 후크** — 본격 양방향 루프 (다소 무거움)
6. **C.3-4 단톡방 그룹 결과** — 가장 강력하지만 가장 무거움 (장기 과제)

## 한 줄 결론

**우리는 "결과 페이지를 만든다"가 아니라 "친구한테 보내는 한 컷을 만든다"고 봐야 한다.** 토스의 디자인 원칙은 그 한 컷을 만드는 도구.

캡처되지 않는 결과 페이지 = 트래픽 0 = 광고 수익 0.
