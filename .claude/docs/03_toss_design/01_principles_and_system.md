## A. 5대 디자인 철학

### A.1. "Easy to Answer" — 3초 룰

> "3초 안에 답이 안 나오면 어려운 질문이에요." — 김재현 Design Strategy Lead [S1]

**원칙**: 사용자가 화면을 보고 3초 안에 다음 행동을 떠올릴 수 없으면 그 화면은 실패.

**토스 사례**: 보험금 청구 첫 화면에 "전문가 도움받기 / 직접 신청하기" 두 카드를 띄웠더니 이탈률 폭증. **"서류 있어요?"** 단일 질문 한 줄로 바꾸자 이탈률 약 50~60% 감소.

**우리 적용**: 결과 페이지 최상단 첫 줄은 "당신은 INFP입니다" 같은 카테고리 레이블이 아니라 **"이거 나잖아"가 즉시 자극되는 킬러 한 줄**이어야 한다. 유형명은 그 다음.
- ✗ "INFP — 중재자형, 당신의 결과를 안내드립니다"
- ✓ "너 머릿속 너무 시끄럽지 않아?" → 그 아래 "INFP, 이런 사람이에요"

---

### A.2. "1 Thing / 1 Page"

> 한 화면에 하나의 결정만. 브랜드 로고조차 과감히 생략. — dalgudot [S6], wepostit [S7]

**원칙**: 사용자가 한 화면에서 결정해야 할 것이 두 개면 인지 부하 폭증. 한 화면 = 한 결정.

**토스 사례**: 회원가입을 한 화면 = 한 입력 (이름 → 주민번호 → 통신사 → 휴대폰)으로 분리. 한 페이지에 모든 입력을 몰지 않음. 송금 화면도 "받는 분"·"금액"·"확인"이 각각 다른 화면.

**우리 적용**: 결과 페이지는 하나의 긴 페이지가 아니라 **스크롤 단위로 정보 한 덩어리**다.
- 화면 1: 킬러 라인 + 유형명 + 일러스트 (강한 첫인상)
- 화면 2: 구체 상황 묘사 3~5개 ("회식 끝나고 카톡 답 천천히 보낸다")
- 화면 3: 관계/연애에서의 모습
- 화면 4: 같이 놀면 좋은 유형 / 피곤한 유형
- 하단 fixed: "친구한테 보여주기" (Always)

### A.3. 첫 화면 임팩트 — "움직이거나, 수가 많거나, 호기심을 자극하거나"

> "첫 화면에서 승부를 봐야 한다, 3초 안에 결정되는 첫인상을 임팩트 있게 남겨야 한다." — 토스 브랜드 디자이너 [S5]

**강력한 첫인상의 3가지 도구**:
1. **움직임** — 스크롤로 하늘이 상승하거나, 화면이 어두워지는 메타포
2. **숫자의 힘** — 큰 수치 한 개를 압도적 크기로
3. **호기심 자극** — "?"를 일으키는 비주얼이나 문장

**우리 적용**: 결과 페이지 첫 화면에 일러스트/이모지 1개 + 킬러 라인 1개 + 유형명. 셋 다 강해야 함.
- 일러스트는 캐릭터/감정 1컷 (정적 OK, 모션 가능하면 더 좋음)
- 킬러 라인은 호기심·공감 자극형 ("너 머릿속 너무 시끄럽지 않아?")
- 유형명은 4글자 + 한 줄 별명 ("INFP, 머릿속 우주 만드는 사람")

---

### A.4. 잡초 제거 3단 위계 — Customer Centric의 운영 룰

> 토스팀 라이팅 프린시플은 **3단 잡초 제거** 구조 [S3]:
> 1. **Weed cutting** — 문장 안의 잡초 (의미 없는 단어 제거)
> 2. **Remove empty sentences** — 화면 안의 잡초 (의미 없는 문장 제거)
> 3. **Focus on key message** — 화면 간의 잡초 (한 화면당 핵심 메시지 1개로 압축)

**우리 적용**: 결과 카피를 쓸 때 위에서 아래로 3단 검수.

| 단계 | 검수 질문 | MBTI 결과 페이지 사례 |
|---|---|---|
| 1. 문장 안 잡초 | 빼도 의미 안 변하는 단어가 있는가? | "당신께서는 매우 깊은 감수성을 가지신 분이라고 할 수 있습니다" → "당신은 감수성이 깊어요" |
| 2. 화면 안 잡초 | 빼도 메시지 안 변하는 문장이 있는가? | "결과를 확인해주세요. 결과는 다음과 같습니다." → 첫 줄 통째로 삭제 |
| 3. 화면 간 잡초 | 한 화면에 핵심 메시지가 2개 이상인가? | "당신은 감수성이 깊고, 인간관계에 진심이며, 일에서는 완벽주의" → 3개 화면으로 쪼개기 |

---

### A.5. 모션의 3대 목적 — 정보 전달 / 감정 표현 / 사용자 참여

> "100마디 말보다 1번의 움직임" — 토스피드 [S8]

**원칙**: 모션은 예쁘기 위해서가 아니라 **3가지 목적 중 하나**일 때만 쓴다.

1. **정보 전달** — 페이지 전환, 데이터 변화 시각화
2. **감정 표현** — 신용점수 상승·월급날·생일에 폭죽 애니메이션 / 불편한 상황엔 가벼운 애니메이션으로 무게감 완화
3. **사용자 참여** — 사용자가 움직임의 주체가 되는 인터랙티브 그래픽

**우리 적용**: 결과 페이지 모션은 두 군데에서만.
- 결과 등장 시: **감정 보상** — 일러스트가 살짝 튀어오르거나, 유형명이 타이프라이터처럼 등장 (자기 결과를 받는 순간의 작은 보상)
- "친구한테 보여주기" 버튼: **사용자 참여** — 탭 시 살짝 눌리는 spring 효과

자세한 모션 룰은 D 섹션.

## B. UX 라이팅 — 5 코어밸류 + 8 프린시플

### B.1. 5 코어밸류 (토스 공식) [S3, S10]

| 가치 | 정의 | MBTI 결과 페이지 적용 |
|---|---|---|
| **Clear** | 메이커가 아닌 사용자가 이해할 수 있는 문장 | "내향형 직관 감정 인식" 같은 학술 용어 X |
| **Concise** | 꼭 필요한 내용만 필요한만큼 | 결과당 400~800자, 그 이상 X |
| **Casual** | 어려운 용어와 딱딱한 뉘앙스 → 쉽고 친절하게 | "고지" → "알려드려요", "수신" → "받기" |
| **Respect** | 당장의 이익 포기하더라도 진실되고 사용자 존중 | 바넘 효과만 의존하는 두루뭉술 카피 X |
| **Emotional** | 정보 뒤의 감정에 공감 | "당신은 회식 끝나면 진이 빠지죠" 같은 정서 짚기 |

> 외부 분석에서도 **"Casual이 다른 서비스와 토스를 구분짓는 결정적 요소"** [S11].

---

### B.2. 8 라이팅 프린시플 [S3]

1. **Predictable hint** — 다음 화면을 예상할 수 있게
2. **Weed cutting** — 의미 없는 단어 제거
3. **Remove empty sentences** — 화면 내 의미 없는 문장 제거
4. **Focus on key message** — 화면당 핵심 정보 1개
5. **Easy to speak** — 입으로 읽었을 때 자연스럽게
6. **Suggest over force** — 강요 아닌 권유
7. **Universal words** — 전 연령·교육 수준 이해 가능
8. **Find hidden emotion** — 정보 뒤 감정에 공감

---

### B.3. 구체 카피 룰 (앱인토스 공식 가이드 발췌) [S10]

| 룰 | 예시 |
|---|---|
| **해요체 일관** | "~합니다" "~합니다만" 금지 |
| **과도한 경어 금지** | "~시겠어요?" → "~할래요?" / "계시다 → 있다" / "여쭈다 → 묻다" |
| **명사+명사 금지** | "결과 안내 페이지" → "당신, 이런 사람이에요" |
| **다이얼로그 좌측은 [닫기]** | [취소] 금지 (작업 자체 취소로 오해 방지) |
| **긍정형 우선** | "사진을 못 올렸어요" → "사진을 다시 올려볼까요?" |
| **'되어요' → '돼요' 통일** | 모바일 좁은 공간 고려 |
| **능동형 기본** | "결과가 안내되었습니다" → "결과 나왔어요" |

---

### B.4. MBTI 결과 페이지 카피 변환표 — Before / After

토스의 카피 철학을 우리 결과 페이지에 적용한 직접 변환표. 작업할 때 이 표를 옆에 두고 쓴다.

| 상황 | 관습형 (피하기) | 토스식 (쓰기) |
|---|---|---|
| 결과 호명 | "당신의 결과는 INFP입니다" | "INFP, 이런 사람이에요" |
| 결과 도입 | "결과를 안내해드립니다" | (없음 — 잡초) |
| 성격 묘사 | "감수성이 풍부하고 내향적인 성향을 가진 분입니다" | "당신은 머릿속이 늘 시끄러워요" |
| 관계 묘사 | "타인에게 진심을 다하는 경향이 있습니다" | "친한 친구한테는 진심이 너무 깊어서 가끔 본인도 힘들어요" |
| 공유 버튼 | "결과 공유하기" | "친구한테 보여주기" |
| 다른 테스트 | "다른 검사 보러 가기" | "내 친구는 어떤 사람일까?" |
| 로딩 메시지 | "분석 중입니다..." | "당신, 이런 사람이군요" (결과 보여주기 직전) |
| 에러 | "오류가 발생했습니다" | "잠깐, 다시 한번 해볼래요?" |

> **카피 작성 셀프 체크**: 친구한테 카톡으로 보낸다고 생각하고 다시 읽어봐서 어색하면 다시 쓴다.

---

### B.5. 에러 메시지 철학 — "Navigating Error" [S12]

> "에러 메시지의 역할은 다음 화면으로 넘어갈 수 있게 안내하는 것"

에러는 책임 추궁이 아니라 **탈출 경로 제공**. 우리 페이지에서 테스트 도중 오류가 나도 "다시 해볼래요? [다시 시작]" 식으로.

## C. TDS — Toss Design System

> TDS 컴포넌트 코드는 비공개. 우리는 *철학*만 차용.

### C.1. 색상 — 단일 메인 + 채도 변주 [S13]

**토스의 룰**:
- 메인 브랜드 컬러는 1개 (#1F4EF5) — 외부 분석 [S13]
- 색상을 늘리지 않고 **같은 색의 채도/명도 변주**로 위계 표현
- 다크 모드는 별도 명도 기준, 명도 대비를 더 강하게

**우리 적용** (다크 배경 #0a0816 위):
```
배경:        #0a0816    (이미 결정)
본문 텍스트: #f4eed8    (이미 결정, warm white)
보조 텍스트: rgba(244, 238, 216, 0.7)
구분선:      rgba(244, 238, 216, 0.12)

포인트 컬러: 1개만 정한다 (예: #b8a4ff 라벤더 / #ffd166 골드 / #6ee7b7 민트)
  - 변주 1 (강조):      포인트 컬러 100%
  - 변주 2 (중강조):    포인트 컬러 70%
  - 변주 3 (배경 hint): 포인트 컬러 12% 위에 본문
```

→ **포인트 컬러 1개 결정 = 브랜드 결정**. 16개 유형마다 색을 다르게 할지(예: INFP=라벤더, ENTP=오렌지) 아니면 전체 통일할지는 별도 논의 필요.

### C.2. 타이포그래피

**토스의 룰** [S14]:
- 토스 Product Sans (자체 폰트, 비공개)
- 타이포 토큰 13개 (제목 → 본문 → 캡션)
- 사이즈 30px → 11px / line-height 40px → 16.5px
- Weight: Light, Regular, Medium, Semibold, Bold (5단계)

**우리 적용** (Pretendard 사용):
```
Typo 1 (히어로 킬러 라인):  30px / line 40px / Bold
Typo 2 (유형명):             24px / line 32px / Bold
Typo 3 (섹션 제목):          20px / line 28px / Semibold
Typo 4 (본문 강조):          17px / line 26px / Medium
Typo 5 (본문):               16px / line 24px / Regular
Typo 6 (보조):               14px / line 20px / Regular
Typo 7 (캡션):               12px / line 16px / Regular
```

→ Pretendard는 토스 Product Sans와 가장 가까운 대용 [S15]. 토스가 공식 사용하는 폰트는 아니지만 한국 UI 시장 사실상 표준.

### C.3. 스페이싱

토스 공식 명시는 발견 안 됨. 외부 분석 기준 [S13]:
- 메인 화면 좌우 마진 15px, 서브 페이지 22px
- 터치 영역 40~50px (피츠의 법칙 적용)
- **8pt 그리드를 고집하지 않음** — 4pt, 때로는 2pt도 활용 [S16]

**우리 적용**: 4pt 베이스. 컴포넌트 안 간격은 4·8·12·16·24·32·48·64.

### C.4. 컴포넌트 — 차용할 패턴

토스가 사용하는 핵심 컴포넌트 11개 [S17] 중 우리에게 직접 필요한 것:

#### BottomCTA (가장 중요)
- 화면 하단 fixed
- SafeArea 자동 패딩
- 키보드 위에 고정
- **우리 결과 페이지의 "친구한테 보여주기" 버튼**이 정확히 이 패턴

#### ListRow
- 좌측 아이콘 + 가운데 텍스트(2줄까지) + 우측 보조 정보
- 우리 결과 페이지의 "구체 상황 묘사 3~5개"에 적용

#### Sheet (BottomSheet)
- 화면 하단에서 올라오는 모달
- 배경 불투명도 **20%** (원래 화면 인식 가능) — 토스 룰 [S18]
- 우리 결과 페이지의 "공유하기" 누르면 카톡/인스타/링크복사 분기 시트로

#### Alert (중앙 모달)
- 배경 불투명도 **40%** (집중 강제)
- 시스템 오류·긴급 결정에만
- 우리 페이지에선 거의 안 씀

#### Top (헤더)
- 좌측 ← / 가운데 타이틀 / 우측 보조 액션
- 결과 페이지에선 헤더 자체를 빼는 게 토스답다 (1 Thing / 1 Page)

### C.5. 모달 띄우는 룰 [S18]

| 종류 | 위치 | 배경 어둡기 | 언제 |
|---|---|---|---|
| Alert | 화면 중앙 | 40% (집중 강제) | 시스템 설정·파괴적 작업·긴급 |
| Sheet | 화면 하단 | 20% (원화면도 인식) | 다음 화면 진행 전 작업, 현 화면에 영향 |

**Thumb Zone**: Sheet의 Primary Button은 항상 화면 하단(엄지 도달 영역). 이게 자연스러운 진행 흐름.

## D. 모션 / 인터랙션

### D.1. Rally — 토스 자체 모션 라이브러리 [S2]

iOS / Android / Web 공통 인터랙션 라이브러리. **비공개**. Easing/timing 구체 수치도 미공개.

→ 우리는 framer-motion + CSS transition으로 시작. easing은 `cubic-bezier(0.32, 0.72, 0, 1)` 같은 토스풍 spring 곡선 흉내.

### D.2. 인터랙션의 ROI 사례 [S2]

대출 심사 로딩 화면에서 실시간 실제 상품 노출 → **이탈률 감소, 결과화면 도달률 향상**. 인터랙션은 "예쁘게"가 아니라 "**더 명확한 피드백 + 다음 행동 직관 노출**"의 기능적 장치.

→ 우리 페이지의 테스트 진행 중 로딩에도 응용 가능: "당신을 분석 중..." 같은 빈 로딩 X. 대신 "지금까지 답변 보면 INFP·ENFP 사이..." 같은 실시간 힌트 노출.

### D.3. 페이지 전환

토스의 트랜지션 = 모션 그래픽 영상문법의 **트랜지션 포인트** 적용 [S2].

**우리 적용**:
- 테스트 문항 간 전환: 우→좌 슬라이드 (다음 진행), 좌→우 슬라이드 (이전 복귀)
- 테스트 → 결과 전환: 페이드 + 살짝 위로 올라옴
- 결과 페이지 내 섹션 전환: 스크롤 트리거 fade-in (intersection observer)
