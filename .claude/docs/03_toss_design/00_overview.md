## 토스 5대 핵심 원칙 (한 줄 압축)

1. **3초 룰** — 첫 화면에서 3초 안에 답이 안 나오면 어려운 질문이다 [01.A.1]
2. **1 Thing / 1 Page** — 한 화면에 하나의 결정만. 브랜드 로고조차 과감히 생략 [01.A.2]
3. **잡초 제거 3단** — 문장 안 잡초(weed cutting) → 화면 안 잡초(empty sentences) → 화면 간 잡초(focus on key message) [01.A.4]
4. **사용자 시점 질문형 카피** — "이체" 아니라 "어디로 돈을 보낼까요?" [01.B]
5. **모션은 장식 아닌 기능** — 정보 전달·감정 표현·사용자 참여 3가지 목적에서만 [01.D]

## INFP 결과 페이지 한 화면 레이아웃 (ASCII 목업)

```
┌─────────────────────────────────┐
│ #0a0816                         │  ← 다크 배경 (이미 결정됨)
│                                 │
│   너 머릿속                     │  ← 킬러 라인 (Typo 1, 30px, Bold)
│   너무 시끄럽지 않아?           │     "느낌이 맞다" 자극, 3초 룰
│                                 │
│   INFP, 이런 사람이에요         │  ← 유형명, 일상어 호명
│                                 │
│        [감정 일러스트 1개]       │  ← 호기심/움직임 비주얼 1개만
│                                 │
│   ─────────────                 │
│                                 │
│   • 회식 끝나고 카톡 답         │  ← 구체 상황 묘사 3~5개
│     천천히 보낸다               │     (한국 맥락 + 청킹)
│   • 택배 인터폰 두렵다          │
│   • ...                         │
│                                 │
│   ─────────────                 │
│                                 │
│   [친구한테 보여주기]    ← 하단 fixed BottomCTA
│                              엄지 도달 영역, 단일 행동
└─────────────────────────────────┘
   ↑ 인스타 9:16 캡처 시 한 화면에 잡힘
```

세부 매핑은 [01_principles_and_system.md](01_principles_and_system.md)와 [02_share_engine.md](02_share_engine.md) 참조.

## 결과 페이지 작업 체크리스트

작업할 때 이 체크리스트를 끝까지 통과시키자. 하나라도 빠지면 토스 감성도 안 살고 공유도 안 된다.

### 카피
- [ ] 첫 화면 첫 줄이 **3초 안에 "이거 나잖아"** 자극하는 킬러 라인인가? (예: "너 머릿속 너무 시끄럽지 않아?")
- [ ] 모든 문장이 **해요체**인가? ("~합니다" / "~시겠어요?" 금지)
- [ ] **명사+명사 조합 제거**했나? ("자기 분석 결과 안내" → "당신, 이런 사람이에요")
- [ ] 결과 카피에 **"회식·카톡·단톡방·택배·인터폰" 같은 한국 일상 맥락** 단어가 들어갔나?
- [ ] 한 섹션에 핵심 메시지 **1개**만 있는가? 두 개면 쪼개기.
- [ ] 캡처 1장으로 잘릴 수 있는 화면에 **한 줄 요약**이 반드시 보이는가?

### 레이아웃
- [ ] 메인 컬러 **1개**로 통일했는가? (다크 배경 #0a0816 위 포인트 컬러 1색)
- [ ] 같은 컬러의 **채도/명도 변주만으로** 위계를 만들었는가? (색 종류 늘리지 말 것)
- [ ] 하단에 **고정 BottomCTA** 1개 ("친구한테 보여주기")가 있는가?
- [ ] 엄지 도달 영역(화면 하단 1/3) 안에 주요 액션이 있는가?
- [ ] 1:1 비율로 캡처해도 **킬러 라인 + 유형명 + 일러스트**가 한 프레임에 잡히는가?

### 모션 (있다면)
- [ ] 모션이 **정보 전달·감정 표현·사용자 참여** 중 하나의 목적이 있는가? 예쁘기만 한 모션은 빼기.
- [ ] 결과 등장 시 작은 폭죽/번쩍임 같은 **감정 보상**이 있는가? (토스 신용점수 상승 패턴)

### 공유
- [ ] OG 이미지를 **1:1(인스타 정사각)** 과 **1.91:1(카톡·페이스북 링크)** 두 비율로 만들었는가?
- [ ] OG 이미지에 **킬러 한 줄 + 유형 4글자**가 보이는가?
- [ ] "친구한테 보여주기" 버튼이 **카카오톡 / 인스타 스토리 / 링크 복사** 분기를 가지는가?

세부 가이드는 [02_share_engine.md](02_share_engine.md).

## 적용 우선순위

작업 시간이 한정적이라면 이 순서로:

1. **카피 톤 잡기** (가장 싸고 효과 큼) — UX 라이팅 5 코어밸류 + 8 프린시플 적용
2. **단일 컬러 + 채도 변주** 색 시스템 — 메인 컬러 1개 골라서 토큰화
3. **1 Thing / 1 Page** 레이아웃 — 결과 페이지를 스크롤 단위로 쪼개기
4. **하단 고정 BottomCTA** — "친구한테 보여주기" 항상 노출
5. **OG 이미지 2종 제작** — 캡처와 링크 공유 둘 다 커버
6. (여유되면) 모션 — 결과 등장 시 작은 감정 보상

## 무엇을 의도적으로 안 가져왔나

- **토스 Product Sans 폰트** — 비공개. 우리는 [Pretendard](https://github.com/orioncactus/pretendard) 그대로 사용 (이미 결정됨).
- **TDS의 OKLCH 색공간 시스템** — 우리 규모에 과함. 메인 컬러 1개 + Tailwind/CSS 변수 채도 변주로 충분.
- **Rally 모션 라이브러리** — 비공개. CSS transition + framer-motion 정도로 시작.
- **TDS 컴포넌트 코드** — 비공개. 컴포넌트 *철학*만 차용 (BottomCTA, ListRow 패턴 등).

## 한 줄 결론

**우리 결과 페이지가 "토스처럼 보이는가?"가 아니라, "캡처해서 친구한테 보내고 싶은가?"로 검증한다.** 토스 원칙은 그 검증을 통과시키기 위한 도구일 뿐.
