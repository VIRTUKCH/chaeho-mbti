# 색 팔레트 — chaeho-mbti

> chaeho-mbti의 색은 이 파일에 정의된 토큰만 사용한다. CSS·JSX·svg 어디에도 hex 직박 금지.

---

## 원칙

- **단일 메인 컬러 + 채도 변주** — 색의 종류를 늘리지 말고, 한 액센트의 채도/명도로 위계 표현.
- **라이트 모드 1차** — 다크 모드는 미정의. 도입 결정 시 같은 토큰 이름에 다른 hex만 매핑.
- **WCAG AA 콘트라스트 통과** — 본문 4.5:1 이상, 대형 텍스트 3:1 이상.
- **단순함이 깔끔함** — 한 화면에 동시 사용 토큰 8개 이내 권장.

---

## 토큰

### 표면 (Surface)

| 토큰 | hex | 용도 |
|---|---|---|
| `--bg` | `#ffffff` | 페이지 배경 |
| `--bg-surface` | `#f7f7f8` | 카드, 시트, 표면 |
| `--bg-muted` | `#f0f0f3` | 비활성 영역, subtle hint |

### 텍스트

| 토큰 | hex | 콘트라스트 (vs `--bg`) | 용도 |
|---|---|---|---|
| `--text` | `#18181b` | 16.7:1 | 본문 본체 |
| `--text-secondary` | `#52525b` | 8.8:1 | 보조 텍스트, 캡션 |
| `--text-muted` | `#a1a1aa` | 2.8:1 | placeholder, 비활성 (대형 텍스트에만) |
| `--text-on-accent` | `#ffffff` | — | 액센트 면 위 텍스트 |

### 구분선

| 토큰 | hex | 용도 |
|---|---|---|
| `--border` | `#e4e4e7` | 카드·입력 외곽선 기본 |
| `--border-strong` | `#d4d4d8` | hover·focus 강조 |

### 액센트 (인디고 — 단일 메인)

| 토큰 | hex | 용도 |
|---|---|---|
| `--accent` | `#4f46e5` | 주요 액션 버튼, 링크 |
| `--accent-hover` | `#4338ca` | hover·active |
| `--accent-bg` | `#eef2ff` | 선택 상태 배경, soft hint |

### 피드백 (필요할 때만)

| 토큰 | hex | 용도 |
|---|---|---|
| `--success` | `#16a34a` | 성공 메시지, 체크 |
| `--danger` | `#dc2626` | 에러, 파괴적 작업 경고 |

---

## CSS 변수 (복붙용)

```css
:root {
  /* 표면 */
  --bg: #ffffff;
  --bg-surface: #f7f7f8;
  --bg-muted: #f0f0f3;

  /* 텍스트 */
  --text: #18181b;
  --text-secondary: #52525b;
  --text-muted: #a1a1aa;
  --text-on-accent: #ffffff;

  /* 구분선 */
  --border: #e4e4e7;
  --border-strong: #d4d4d8;

  /* 액센트 */
  --accent: #4f46e5;
  --accent-hover: #4338ca;
  --accent-bg: #eef2ff;

  /* 피드백 */
  --success: #16a34a;
  --danger: #dc2626;
}
```

`02_fe_react/src/styles/global.css`의 `:root` 블록에 이걸 그대로 박아 두고, 컴포넌트는 `var(--token)`으로만 참조한다.

---

## 사용 규칙

- 모든 색은 **`var(--token)` 형태**로만 사용한다.
- 그라데이션이 정말 필요하면 토큰 두 개의 보간으로:
  ```css
  background: linear-gradient(135deg, var(--accent), var(--accent-hover));
  ```
- 반투명이 필요하면 토큰 hex를 풀어쓰지 말고 동일 토큰의 alpha 버전을 *이 파일에 새 토큰으로 추가*해서 쓴다. 임시 `rgba(79, 70, 229, 0.08)` 금지.
- 새 시맨틱 컬러(warning, info 등)가 필요해지면 **이 파일에 먼저 추가**한 뒤 사용. 그 전에는 쓰지 말 것.
- 토큰 추가/변경은 항상 이 파일이 단일 출처 (Single Source of Truth).

---

## 안 쓰는 것

- **다중 액센트 컬러** (파랑 + 빨강 + 초록 동시) — 정보 위계 무너짐.
- **페이지 단위 그라데이션 배경** — 캡쳐 시 색이 깨지고, 인쇄·접근성 대응 어려움. 결과지 캡쳐가 핵심 자산이므로 단색 배경이 안전.
- **다크 톤 페이지 배경** (`#0a0816`, `#0e0a1f` 등) — 이 팔레트는 라이트 1차. 다크 모드 도입 결정 전에는 사용 금지.

## 한 줄 결론

> **"이 토큰 안에서만 한다"**가 깔끔함의 본질. 늘리고 싶을 때마다 이 파일에 추가하기 전에 *정말 새 토큰이 필요한지* 다시 묻는다.
