# 결과 페이지 전략 — chaeho-mbti

> 다른 AI/사람이 이 문서만 보고도 16개 결과 페이지를 동일한 톤·구조·디자인으로 만들 수 있게 하는 명세.
> 근거: [.claude/docs/04_viral/](../../../docs/04_viral/) 리서치 (5단 카피 공식 + 한국식 공감 메커니즘).

---

## 1. 이 페이지가 존재하는 이유 (절대 잊지 말 것)

> **결과 페이지 = 공유 엔진 = 프로젝트 매출 엔진.**

- 사용자가 결과를 받으면 그 자체로 끝나는 게 아니라 **카톡/인스타에 캡쳐 박는 행위**가 일어나야 한다. 그게 viral 동력.
- 모든 디자인·카피 결정의 기준은 단 하나: **"사용자가 이 화면을 캡쳐해서 친구한테 보낼 때 한 줄 보태고 웃을 수 있는가?"**
- "정확한 진단"이 목표가 아니다. **"내가 일상 카톡에서 농담으로 쓰던 자기서사를 누가 대신 라벨링해주는 경험"**이 목표다.
- 진지·교훈적 라이프코칭 톤 = 즉시 망함. 자기조롱 가능한 밈 라벨 = 살아남음.

---

## 2. 페이지 구조 (위에서 아래로, 이 순서 그대로)

모든 유형이 이 12블록 구조를 똑같이 따른다. 콘텐츠만 다르고 골격은 동일.

```
[1]  헤더 — 작은 태그("채호 MBTI · 결과") + 큰 4글자 코드("I N F P")
[2]  호명형 제목 — 한 줄, 한 호흡. "머릿속이 너무 시끄러운 사람" 류
[3]  킬러 한 줄 (인용 박스, 금색 강조) — 캡쳐 떴을 때 가장 먼저 박히는 한마디
[4]  요약 단락 — 3~4문장. 겉/속, 평소/특정상황 같은 양면 단정 포함
[5]  시나리오 5개 — "아마 이런 적 있을 거야"
     각 시나리오 = 이모지 + 짧은 라벨(예: "카톡 답장") + 한 문장(구체 장면)
     추상 형용사 금지. "외향적이다" X / "30초 동안 사과 시나리오 다섯 개 가동" O
[6]  잘못 듣는 말 3개 — "주변에서 자주 듣는데, 사실은 다른 말"
     형식: `"외부 평가" (실제는 ~)`. 양면 단정 강화.
[7]  위로형 reframe — "그래도 다행인 건"
     단점을 강점으로 뒤집어주는 한 단락. 자기연민으로 끝나지 않게 들어 올려준다.
[8]  궁합 매치 3개 — "너랑 합이 잘 맞는 사람"
     good 2개 + danger 1개. 친구 호출 동선의 핵심.
     각 매치 = 이모지 + MBTI 4글자 + 한 문장 설명.
[9]  잠긴 64유형 — "잠긴 결과 · 64유형 상세"
     "곧 공개". 호기심 유발 + 재방문 동기.
[10] 공유 캡션 인용 — "친구한테 이렇게 보내봐"
     사용자가 그대로 카톡에 복붙할 한 줄을 표면에 노출.
[11] CTA 2개 — Primary: 친구한테 보내기 · "이거 너잖아" / Secondary: 다음 유형 보기 →
[12] Footer — "chaeho-mbti · 한국식 공감 성격 테스트"
     캡쳐 이미지에 자연스러운 워터마크 역할.
```

블록 추가·삭제·순서 변경 금지. 카피만 갈아끼운다.

## 3. 카피 작성 규칙 (가장 중요 — 작가 역량이 viral과 묻힘의 경계)

### 3-1. 공감의 정의

> **"내가 일상 카톡에서 농담으로 쓰던 자기서사를 누가 대신 라벨링해주는 경험"**

- T발놈이 왜 터졌나: 욕설을 자기조롱 가능한 라벨로 번역해줬기 때문.
- INFP의 "머릿속이 너무 시끄러운 사람"이 박히는 이유: 본인이 카톡에서 농담으로 쓰던 자기서사를 한 줄로 정리해줬기 때문.

### 3-2. 톤 가드레일

| ✅ 무조건 한다 | ❌ 절대 안 한다 |
|---|---|
| 단점을 자기조롱 가능한 밈으로 번역 | 진지·교훈적 라이프코칭 톤 ("당신은 자신을 사랑해야 합니다") |
| 양면 단정 ("겉으론 ~인데 속은 ~") | 한 면만 강조해서 "나는 그런 사람 아닌데" 반발 유발 |
| 구체적 일상 디테일 ("새벽 1시 이불킥") | 추상 형용사 동의 요구 ("당신은 외향적입니다") |
| 본인이 본인 결과 캡쳐해서 웃을 수 있는 라벨 | 외부에서 차별 도구로 쓰일 라벨 ("이런 사람은 결혼 X") |
| 한국어 입말 (~음, ~잖아, ㅋㅋ, ㅠ) | 번역체·논문체 |

### 3-3. 길이 가이드

- 호명형 제목: 1줄 (12~18자)
- 킬러 한 줄: 2줄, 줄당 14~20자. 운율 의식해서 대구 만들기 ("남이 한 한마디로 일주일을 살고, / 본인이 한 한마디로 한 달을 자책한다").
- 요약: 3~4문장 (150~200자)
- 시나리오 텍스트: 한 문장당 30~50자. 5개 모두.
- 잘못 듣는 말: 3개. 한 줄당 15~25자.
- reframe 본문: 2~3문장 (80~140자).
- 매치 설명: 한 문장당 25~40자.
- 공유 캡션: 한 줄, 25~35자, 끝에 ㅠ/ㅋㅋ/소름 같은 입말 감정 토큰 1개.

## 4. 시각 디자인 명세

### 4-1. 캡쳐 친화 원칙

- **모바일 9:16 비율 카드**가 결과 페이지의 시각 단위.
- 모바일 한 스크롤에 [헤더 + 제목 + 킬러 + 요약]이 다 들어가게 한다. 이 영역이 캡쳐의 1순위 대상.
- 카드 최대 너비: 420px. 좌우 패딩: 18px.

### 4-2. 색 팔레트 (고정, 16개 유형 동일)

> 색 토큰은 [@/.claude/ai-context/front_end/color/palette.md](../color/palette.md)가 단일 출처. 여기에는 결과 페이지 의미별 매핑만 기록. **hex 직박 금지, `var(--token)`만**.

| 의미 | 토큰 | 용도 |
|---|---|---|
| 페이지 배경 | `--bg` | 카드 바깥 전체 (따뜻한 크림) |
| 박스 면 | `--bg-surface` | 시나리오·매치 카드, 그리드 셀 |
| 강한 박스 면 | `--bg-muted` | reframe, locked 박스 |
| 강조 면 (액센트 위 텍스트) | `--accent` + `--text-on-accent` | killer 박스, primary 버튼 |
| 강조 텍스트 | `--accent` | 코드(4글자), 태그, 섹션 타이틀, 매치 타입 |
| 본문 텍스트 | `--text` | 제목, 시나리오 본문, reframe 본문 |
| 보조 텍스트 | `--text-secondary` | 요약, locked 본문, 그리드 tagline |
| 약한 텍스트 | `--text-muted` | pager 인덱스, footer 워터마크 |
| 구분선 (약함) | `--border` | 박스 외곽선 기본 |
| 구분선 (강함) | `--border-strong` | dashed 박스 (misheard, shareQuote), secondary 버튼 |
| danger 매치 | `--danger` | 합 안 맞는 매치 텍스트/외곽선 |

색은 유형마다 바꾸지 않는다. 일관성이 사이트 인식의 핵심. **인디고·차가운 톤 회피, 따뜻한 모카 베이스**.

### 4-3. 폰트/타이포

- 시스템 폰트 스택 (한국어 가독성 우선). 별도 웹폰트 없음.
- 코드(4글자): 28px, 800, letter-spacing 0.12em.
- 제목: 30px, 800, letter-spacing -0.02em.
- 킬러 박스: 17px, 600, line-height 1.55.
- 요약/시나리오 본문: 14.5~15px, line-height 1.6~1.7.
- 섹션 타이틀("아마 이런 적 있을 거야" 등): 13px, 700, letter-spacing 0.16em, 라벤더색.

### 4-4. 공유 CSS 모듈

> 16개 유형이 **단일 CSS 모듈** `src/results/result-card.module.css`를 공유한다.

- 페어 구조(.tsx + .module.css per 유형)는 의도적으로 폐기 — 시각 일관성이 viral 인식에 더 결정적이고, 32개 파일의 같은 CSS 복붙이 유지보수 악몽이라.
- 한 유형 = 한 `.tsx` 파일 (콘텐츠만). 시각은 공통 모듈에서.

## 5. 16개 유형 한 줄 컨셉

각 유형의 제목·킬러 방향. 카피 작가가 이 컨셉으로 12블록을 채운다.

| 유형 | 호명형 제목 (제안) | 핵심 자기서사 |
|---|---|---|
| INFP | 머릿속이 너무 시끄러운 사람 | 남이 한 한마디로 일주일, 본인 한마디로 한 달 자책 |
| INFJ | 표정 하나 안 바꾸고 다 알아채는 사람 | 분위기 읽고 본인 감정은 마지막에 확인 |
| INTJ | 회의 중에 이미 다음 분기 짜고 있는 사람 | 효율 안 나오는 자리 못 견딤 |
| INTP | "근데 그거 왜 그래?"가 입버릇인 사람 | 결론보다 과정 분해가 재밌음 |
| ISFP | 말없이 분위기 맞춰주다 혼자 지치는 사람 | 본인 취향은 강한데 말 안 함 |
| ISFJ | 다 챙겨주고 본인 거는 못 챙기는 사람 | 거절 못 해서 새벽까지 일함 |
| ISTP | 5분 전 처음 본 기계도 그냥 고치는 사람 | 말로 설명하기 귀찮음 |
| ISTJ | 약속 10분 전 도착, 5분 전부터 화나는 사람 | 약속 어기는 사람 = 인성 파산 |
| ENFP | 초면에 인생 얘기 다 듣고 오는 사람 | 호기심 폭발, 디테일 마무리 약함 |
| ENFJ | 분위기 안 좋으면 본인 책임인 줄 아는 사람 | 모임 끝나고 혼자 복기 |
| ENTP | 토론 이기려고 본인 의견 반대로 바꾸는 사람 | 논리 게임이 곧 애정 표현 |
| ENTJ | 친구 모임 짤 때 엑셀 켜는 사람 | 비효율 못 봐줌 |
| ESFP | 슬픈 일 있어도 일단 노래방부터 가는 사람 | 우울할수록 사람 많은 곳 |
| ESFJ | 단톡방 안부 인사 매일 첫 빠따인 사람 | 친구 생일 본인이 더 챙김 |
| ESTP | "일단 해보자"로 시작, "어쩌지"로 끝내는 사람 | 후회보다 액션 |
| ESTJ | "결론이 뭔데?"로 회의 5분 단축시키는 사람 | 비효율 = 도덕적 문제 |

각 유형의 매치(궁합)·시나리오·잘못 듣는 말은 위 컨셉을 확장해서 채운다. 단, **3-2 톤 가드레일은 무조건 지킨다**.

## 6. 라우팅·네비게이션 동선

### 6-1. 16유형 순환

- URL 쿼리 파라미터로 분기: `?type=INFP`, `?type=ENTJ` 등.
- 파라미터 없거나 잘못된 값이면 INFP로 fallback.
- App.tsx에서 type → 컴포넌트 매핑 객체 + 다음/이전 인덱스 계산.

### 6-2. CTA 배치

- Primary("친구한테 보내기"): 공유 액션 (navigator.share → clipboard 폴백).
- Secondary("다음 유형 보기 →"): 인덱스 +1 (16번째 누르면 1번째로 순환).
- 화면 하단에 작게 "← 이전 / 다음 →" 페이지네이션도 같이 두어서 16개 훑기 부드럽게.

라우터 라이브러리(react-router 등) 도입 금지 — 결과지 검증 단계라 의도적으로 가볍게 간다.

## 7. 공유 동선

### 7-1. 동작

```ts
const handleShare = async () => {
  const text = `${shareCaption} 너도 해봐 👇`
  if (navigator.share) {
    await navigator.share({ title, text, url })
    return
  }
  await navigator.clipboard.writeText(`${text} ${url}`)
  alert('링크를 복사했어. 친구한테 붙여넣기!')
}
```

- 모바일에서 navigator.share = OS 공유 시트.
- 데스크탑/미지원 환경 = 클립보드 복사 + alert.

### 7-2. 캡션 노출

- 화면에도 같은 캡션을 인용 박스로 노출해서 사용자가 캡쳐할 때 자동으로 같이 박히게 한다.
- 캡션 1줄, 끝에 입말 감정 토큰 1개(ㅠ/ㅋㅋ/소름 등).

## 8. 워터마크 — 캡쳐 떠도 우리 사이트가 박힌다

- footer 마지막 줄: `chaeho-mbti · 한국식 공감 성격 테스트 (개발 중)`
- 회색·작은 글씨로 카드 하단에 고정. 캡쳐 이미지 어디까지 자르든 대부분 같이 박힌다.
- "(개발 중)" 텍스트는 정식 출시 시 제거.

## 9. 출시 전 검증 (생존자 편향 회피)

리서치 라운드 2.5 강한 반박: 5단 공식을 따랐다고 viral이 보장되지 않는다. 패턴 모방으로 끝내지 말 것.

- **베타 N=20~30**: "이 결과가 내 얘기 같다" (1~5점) + "친구한테 보내고 싶다" (1~5점). **두 평균 4점 미만이면 카피 재작성**.
- 결과별 공유 버튼 클릭율 추적. 약한 유형 식별 → 카피 다시.
- 안티 패턴 체크:
  - [ ] 결과 페이지 진지·교훈적 톤 없음
  - [ ] 외부에서 차별 도구로 쓰일 라벨 없음
  - [ ] 결과 URL이 유형별로 고유함 (`?type=XXXX`)
  - [ ] 모바일 한 스크롤에 핵심 다 들어감
  - [ ] 결과 직후 큰 공유 CTA
  - [ ] 친구 호출 동선(매치) 존재
  - [ ] footer 워터마크 존재

## 10. 한 줄 결론

> **이 페이지에서 차별화는 외형이 아니라 카피 톤에서 난다.** 12블록 골격·색 팔레트·라우팅은 표준대로. 모든 비싼 시간은 **유형별 카피 한 줄 한 줄**에 쓴다. 그게 viral과 묻힘의 경계다.
